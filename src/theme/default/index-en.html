Multi-level proxy forwarding

For each data connection, 2-4 servers across multiple countries are used to forward the network connection to ensure the non-retrieving of the source IP, and safety first

Dynamic Transmission Protocol

Each data connection uses a dynamic transmission protocol connection (self-developed data transmission protocol). The protocol has no fixed characteristics, cannot be identified, and safety first.

Multi-channel data distribution

Multi-channel recurrence technology is used to ensure the stability and reliability of data transmission, and the connection will not be interrupted due to relay server failure (IP blocked, server failure, etc.)

Dynamic data encryption

Multi-layer encryption, fixed and dynamic keys are combined to ensure the security of the data transmission process and cannot be cracked

Network independent key

Each network created by the user has an independent encryption and decryption key to ensure the isolation between the network and the network

Stable without fear of the wall Not afraid of being intercepted

The newly developed dynamic transmission protocol and multi-channel distribution technology, using LightNet VPN stability to fully upgrade, is a real VPN that breaks through the firewall, and is much better than the open source technology solutions on the market

Virtual network concept Facilitate team management and sales

LightNet VPN abstracts the concept of a layer of network. The network is composed of a network number and a serial number. A network has multiple serial numbers, and a unique network certificate is composed of 1 network number + 1 serial number. The owner of the network can manage and sell the serial numbers below.

SaaS architecture VPN

This is a SaaS-based VPN product. Each user can create a dedicated network, manage his own serial number under the network, and sell it to customers or team members. Sales and maintenance of sold products
